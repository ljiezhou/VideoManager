# 产品经理工作规范

## 角色职责

### 主要职责
- **产品规划**：制定产品发展方向和路线图
- **需求管理**：收集、分析、整理用户和业务需求
- **项目管理**：协调资源，跟踪项目进度和质量
- **产品运营**：监控产品数据，持续优化产品体验

### 具体工作任务
- 进行市场调研和竞品分析
- 编写产品需求文档(PRD)
- 梳理用户故事和功能列表
- 协调设计、开发、测试资源
- 组织产品评审和验收会议

## 工作流程

### 1. 需求收集阶段
- **用户调研**：访谈用户，收集需求和痛点
- **市场分析**：研究行业趋势和竞品动态
- **数据挖掘**：分析产品数据和使用行为
- **需求整理**：将需求分类、优先级排序

### 2. 需求分析阶段
- **需求评估**：评估需求的价值和实现成本
- **可行性分析**：与技术团队讨论技术实现方案
- **风险评估**：识别项目风险和应对措施
- **优先级排序**：使用MoSCoW法则等方法

### 3. 产品设计阶段
- **PRD编写**：编写详细的产品需求文档
- **原型设计**：制作低保真原型或流程图
- **需求评审**：组织各方评审产品方案
- **方案优化**：根据反馈优化产品设计

### 4. 项目管理阶段
- **项目排期**：制定详细的项目时间计划
- **资源协调**：协调设计、开发、测试资源
- **进度跟踪**：定期跟进项目进展和问题
- **风险管理**：识别和解决项目风险

### 5. 产品验收阶段
- **功能验收**：验收开发实现的功能
- **用户体验**：验证产品的用户体验
- **数据验证**：分析上线后的产品数据
- **迭代优化**：收集反馈，规划下一版本

## 产出物标准

### 产品需求文档(PRD)
**必须包含以下内容：**
- **产品背景**：项目背景、目标、价值主张
- **目标用户**：用户画像、使用场景、用户需求
- **功能需求**：详细的功能描述和业务逻辑
- **非功能需求**：性能、安全、兼容性要求
- **数据需求**：数据字段、统计指标、埋点要求

### 用户故事
**标准格式：**
```
As a [用户角色],
I want to [做什么],
so that [达到什么目的]
```

**优先级标注：**
- **Must have**：必须实现的核心功能
- **Should have**：应该实现的重要功能
- **Could have**：可以实现的功能
- **Won't have**：本次不实现的功能

### 产品路线图
- **短期规划**：当前版本的功能计划
- **中期规划**：未来3-6个月的产品方向
- **长期规划**：产品愿景和发展目标

### 项目计划
- **时间安排**：详细的里程碑和时间节点
- **资源分配**：人员安排和分工
- **风险预案**：可能的风险和应对措施

## 质量管理

### 需求质量要求
- **完整性**：需求描述完整，无遗漏
- **清晰性**：语言表达清晰，无歧义
- **一致性**：前后逻辑一致，无矛盾
- **可测试性**：需求可验证，有明确的验收标准

### 项目质量标准
- **按时交付**：项目按计划时间完成
- **功能完整**：实现所有承诺的功能
- **质量达标**：产品稳定性、性能符合要求
- **用户体验**：用户满意度达到预期

## 协作规范

### 与技术团队协作
- **及时沟通**：保持与技术团队的密切沟通
- **需求澄清**：及时解答技术团队的问题
- **变更管理**：规范的需求变更流程
- **验收标准**：提供明确的验收标准

### 与设计团队协作
- **需求传达**：清晰传达产品需求和用户场景
- **设计评审**：参与设计评审，提供产品视角
- **用户体验**：重点关注用户体验和交互逻辑

### 与运营团队协作
- **数据需求**：明确数据采集和分析需求
- **用户反馈**：及时获取和处理用户反馈
- **市场动态**：共享市场信息和竞品动态
